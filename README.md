# 🚨 Emergency Aid Wallet  

## 🏆Done as a Hackathon Project 

When emergencies strike, **aid must be delivered fast, used properly, and protected from misuse**.  

**Emergency Aid Wallet** is a blockchain-powered system that ensures:  
- Donors can send money securely.  
- Receivers can only use it at **tagged vendors** (e.g., Food, Medicine, Shelter).  
- Funds are **time-bound** → after expiry, the receiver cannot access them.  
- **Trigger refunds manually** if lock expired (automatic refunds can be a future enhancement).  

---

## 💡 Problem
- Aid funds often get **misused or diverted**.  
- Donors lack transparency on **where aid is spent**.  
- Refunds and accountability are **slow and manual**.  

---

## ✅ Our Solution
- 🏷 **Vendor-Tagged Spending** → Receivers can only forward funds to specific vendors.  
- ⏱ **Time-Bound Transfers** → If not claimed in time, funds expire.  
- 🔁 **Manual Refund Button** → Donors reclaim expired funds.  
- 🔒 **Smart Contract Security** → Trustless, transparent execution.  
- 📲 **Simple UI** → Works seamlessly for donors, receivers, and vendors.  

---

## 🔧 How It Works
1. Donor sends funds → smart contract generates **Lock ID + Vendor Tag**.  
2. Receiver claims funds → but can only spend them at that vendor.  
3. Vendor redeems funds.  
4. If time expires → receiver cannot use funds anymore.  
5. **Refund Button** to reclaim money if expired or unused.  

---

## 🏗 Tech Stack
- **Smart Contracts**: Solidity, Hardhat  
- **Frontend**: React.js  
- **Blockchain**: Monad Testnet 
- **Backend**: Node.js scripts  
- **Wallet**: Metamask
